# Blockchain for [Safe Code] – Nova SBE Final Project

## Context
This project was developed as the final assessment for the **Nova SBE Blockchain Academy (5 ECTS)**, an intensive hybrid program on blockchain fundamentals, platforms (e.g., Ethereum / Polygon), smart contracts, governance, and business applications.

## Problem / Question
How can blockchain be applied to **[briefly describe your use case]** in order to improve **[e.g., transparency, traceability, security, data integrity, or efficiency]**?  
The project explores whether a distributed ledger solution is technically and economically viable for this scenario.

## Data and Tools
- Conceptual and secondary data: program materials, public documentation, and industry examples  
- Basic on‑chain data structure design (addresses, transactions, smart contract events)  
- High‑level architecture diagrams for a blockchain-based solution  
- Focus on **blockchain literacy and data flows**, not production‑grade code

## Key Insights and Outcomes
- Mapped the current “as‑is” process and identified main pain points (e.g., information asymmetry, lack of auditability, manual reconciliation).  
- Designed a “to‑be” process using blockchain to record critical events and make data verifiable and tamper‑evident.  
- Defined key entities, on‑chain/off‑chain data separation, and governance assumptions (who writes, who reads, who validates).  
- Proposed measurable KPIs for success (e.g., reduction in disputes, time to verify records, audit effort).  
- Assessed main risks and limitations (scalability, privacy, regulatory constraints) and when a traditional database might still be preferable.
